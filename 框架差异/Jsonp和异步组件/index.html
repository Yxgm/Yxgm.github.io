jsonp和异步组件

jsonp

jsonp：script标签请求到的内容会自动执行，请求回来的回调函数的调用，后端是将把函数的调用和参数一并返回,后端返回的是函数名+jsonString之后数据。浏览器接受到，执行函数，并且将数据自动转化为json对象。

前端请求回来了'test({"name": "sxy"})'，再执行函数，即可完成函数执行

默认的script标签是同步的，也就是当碰到script标签，会请求资源，写在script之前的html会被渲染一遍，等script加载完毕，在渲染后面的html

jquery会在window对象中加载一个全局的函数，当代码插入时函数执行，执行完毕后就会被移除。同时jquery还对非跨域的请求进行了优化，如果这个请求是在同一个域名下那么他就会像正常的Ajax请求一样工作。

补充：script标签动态修改src，脚本会不会执行决定于原来的script标签是否设置了url

如果原来script标签设置了src，在第一次动态修改的时候，将会请求脚本。如果刚开始设置了src，那么就不会请求执行。总结来说，就是在第一次src赋值的时候，才会生效。

异步组件

Code-splitting代码分割，为了减少首屏代码体积，往往会把一些非首屏的组件，一些我们不怎么常用，而且不是太重要的组件，设计成异步组件，按需加载。

异步组件可以减少打包的结果。会将异步组件分开打包(形成不同的chunk)，会采用异步的方式加载组件，可以有效的解决一个组件过大的问题。异步组件的核心可以给组件定义变成一个函数。

异步组件应该是用户的行为预判

react中引入了React.lazy和React.Suspense两个API，配合import语法即可实现异步组件，打包分割和异步加载。

三种异步组件：普通函数异步组件、promise异步组件、高阶异步组件（可配置的）

高级异步组件的实现是非常巧妙的，它实现了 loading、resolve、reject、timeout 4 种状态。异步组件实现的本质是 2 次渲染，除了 0 delay 的高级异步组件第一次直接渲染成 loading 组件外，其它都是第一次渲染生成一个注释节点，当异步获取组件成功后，再通过 forceRender 强制重新渲染，这样就能正确渲染出我们异步加载的组件

webpack 异步加载原理（按需加载）

先判断已经加载的chunk中是否被记载过。

  1. 将 js 模块给独立导出一个.js 文件，记录chunkId，为了后续得到异步加载模块的url地址，打包之后的html中异步组件的link标签是prefetch（也就是预加载，而不是真正请求），之后真正使用模块的时候，动态创建一个 script 对象，加入到 document.head 对象中，浏览器会自动帮我们发起请求。最后用pormise.all一并请求返回，返回chunk.bundle.js文件。

  2. 拿到异步模块的IIFE的chunk.bundle.js文件，此文件被包裹成了(window['webpackJsonp'].push([chunkId],{code...}))格式，，执行IIFE函数，⚠️异步加载的模块就保存在了全局变量webpackJsonp中。push方法重写，push之后，并会执行webpackJsonpCallback函数

  3. webpackJsonpCallback加载异步模块并且安装，并且执行script的回调函数，缓存异步模块，同时将onerror和onload方法置空，防止内存泄露。

  4. 加载的同时，挂载组件，完层渲染。
