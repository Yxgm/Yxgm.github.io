可视化领域

SVG : 自适应特别好，每一个都是一个dom，性能下降。

CANVAS：大数据量下：页面性能好， 使用sava/restore

HTML5 提供了 Canvas 和 SVG 两种绘图技术，也是多数 Web 图表库使用的渲染技术。Canvas 是基于脚本的，通过 JavaScript 指令来动态绘图。而 SVG 则是使用 XML 文档来描述矢量图。两者有不同的适用场景。

canvas和svg的区别

  • canvas绘画出来的图形一般成为位图，也就是放大缩小的情况下会出现失真的情况，svg绘制的图形是矢量图，不存在失真的情况
  • canvas绘制的图形不会出现在DOM结构中，svg绘制的会存在于DOM结构
  • canvas类似于动画，每次图形的改变都是先清除原来的图形，然后把新的图形画上去，svg则是可以直接通过js来进行某些操作
  • canvas依赖于分辨率，svg不依赖分辨率
  • canvas最适合图像密集型的游戏，其中的许多对象会被频繁重绘，svg不适合游戏应用
  • canvas的性能更好，适合大量数据的渲染，canvas的性能收到画布尺寸影响更大，而svg的性能受图形元素个数影响更大。
