基础概念

1. type和interface

重点：用interface描述数据结构，用type描述类型

不同之处

  1. type（类型别名）:侧重于描述数据，可以使用运算符（& ｜）来实现联合类型或者交叉类型。

    联合类型（一次只能使用一个类型，或的关系 |），或者交叉类型（每次是合并后的类型且的关系 &）

  2. interface：重于描述数据结构，使用 extends 来继承 type 或者 interface 来拓展自己的类型。

  3. type不可以重复声明，interface重复声明会合并。

  4. typeof 可以基于一个已有的函数或者对象来获得这个类型

  5. interface 定义之后，如果有范型继承接口，或者别的接口继承，一定要实现，否则会报错，在子容器中一定要实现继承的属性

相同之处
  1. 都可以描述一个对象或者函数
  2. 都允许拓展

2. 断言和约束

  1. 类型断言：缩小类型，明示 ts 编译器，这个变量的类型我是明确知道的，绕过 ts 的类型检查
  2. 类型约束的时候，实参传入函数的时候，必须要保证传递明确声明的类型，（extends 等），也可以传入一些没有定义的类型

补充：对于类型接口的交叉或者联合，相同的基本属性也是对应的交叉或者联合；引用类型会合并操作

3. unknown和any

​ unknown 和 any 的主要区别是 unknown 类型会更加严格：在对 unknown 类型的值执行大多数操作之前，我们必须进行某种形式的检查。而在对 any 类型的值执行操作之前，我们不必进行任何检查,unknown类型是类型安全的any类型，unknown类型，ts会对它检查

4. 双向协变

  • 协变： 允许子类型转换为父类型
  • 逆变： 允许父类型转换为子类型
  • 双向协变：既可以协变又可以逆变，在 TypeScript 中，由于灵活性等权衡，对于函数参数默认的处理是 双向协变 的
  • 不变：不允许变型

5. 类型保护

  1. instanceof
  2. in 判断属性是否在对象之中
  3. typeof

6. 类型兼容

  1. 对象：更具体的类型可以赋值给更宽泛的类型，成员少的可以兼容成员多的

  2. 函数：

​ X兼容Y：X（目标类型）= Y（源类型）

  1. 当一个类型Y可以赋值给另一个类型X，那么就可以说类型X兼容类型Y

  2. 口诀：结构之间兼容：成员少的兼容成员多的 ； 函数之间：参数多的兼容参数少的

  • 在类型系统中，属性更多的类型是子类型，联合类型
  • 在集合论中，属性更少的集合是子集，交叉类型

7. 函数重载

返回值必须是函数实现的子集，或者超集。TS 在检查一个函数重载的返回值与其实现的返回值是否兼容时，是直接看整体的。

  1. 如果函数重载无返回值，兼容
  2. 如果函数重载的返回值是函数实现的返回值的子集，兼容。
  3. 如果函数重载的返回值是函数实现的返回值的超集，也兼容。
  4. 如果说返回的是字面量类型，它不作为string|number 等，可能我们需要 as 类型缩小一下

8. 泛型的理解

​ 泛型程序设计（generic programming）是程序设计语言的一种风格或范式。泛型允许程序员在强类型程序设计语言中编写代码时使用一些以后才指定的类型，在实例化时作为参数指明这些类型。

​ 泛型是指在定义函数，接口或者类的时候，不预先定义好具体的类型，而在使用的时候在指定类型的一种特性，同时可以起到约束某个数据结果的类型的作用。
