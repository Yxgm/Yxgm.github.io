Vue编译原理

解析器（parser）的作用是将 模板字符串 转换成 element ASTs。

优化器（optimizer）的作用是找出那些静态节点和静态根节点并打上标记。

代码生成器（code generator）的作用是使用 element ASTs 生成 render函数代码（generate render function code from element ASTs）。

1. 将模版字符串转化成element AST

​ 主要思想是将该模版字符串扔到一个while循环中，然后一段一段的截取，把截取到的每一小段字符串进行解析，直到最后截没。正则匹配出开始标签，然后记录下来标签名，再去截取标签的attrbute，存储到该标签对象上的attrsList，然后在截取子节点，循环下去。

​ 同时要维护一个 stack。这个 stack 是用来记录一个层级关系的，用来记录DOM的深度。

​ 更准确的说，当解析到一个 开始标签 或者 文本，无论是什么， stack 中的最后一项，永远是当前正在被解析的节点的 parentNode 父节点。

​ 通过 stack 解析器就可以把当前解析到的节点 push 到 父节点的 children 中。也可以把当前正在解析的节点的 parent 属性设置为 父节点。

​ 但并不是只要解析到一个标签的开始部分就把当前标签 push 到 stack 中。因为在 HTML 中有一种 自闭合标签，比如 input。这种自闭合的标签是不需要push到stack中的，因为input并不存在子节点。

​ 所以当解析到一个标签的开始时，要判断当前被解析的标签是否是自闭合标签，如果不是自闭合标签才 push 到 stack 中。

​ 现在有了 DOM 的层级关系，也可以解析出DOM的 开始标签，这样每解析一个 开始标签 就生成一个 ASTElement (存储当前标签的attrs，tagName 等信息的object）

​ 并且把当前的 ASTElement push 到 parentNode 的 children 中，同时给当前 ASTElement 的 parent 属性设置为 stack 中的最后一项。

2. AST树已经生成了，接下来要找出静态节点，打上statci的tag

  1. 先判断自身节点是不是一个静态节点，node.static=isStatic(node);

  2. 然后再循环children，只要children中一个不是静态节点，那么该节点的static将会改变为false，即不是静态节点。

    先解释一下，在上文讲的解析器中将 模板字符串 解析成 AST 的时候，会根据不同的文本类型设置一个 type：

    type 说明
    1 元素节点
    2 带变量的动态文本节点
    3 不带变量的纯文本节点

    所以上面 isStatic 中的逻辑很明显，如果 type === 2 那肯定不是 静态节点 返回 false，如果 type === 3 那就是静态节点，返回 true。

  3. 标记静态节点也就是一个递归的过程，递归遍历一个多叉树

  4. 当前节点是静态节点，并且有子节点，并且子节点不是单个静态文本节点这种情况会将当前节点标记为根静态节点。

    如果发现一个节点是 静态节点，那就能证明它的所有 子节点 也都是静态节点，而我们要标记的是 静态根节点，但是如果一个静态节点只包含了一个文本节点那就不会被标记为 静态根节点。其实这么做也是为了性能考虑，vue 在注释中也说了，如果把一个只包含静态文本的节点标记为根节点，那么它的成本会超过收益~

  5. 整体逻辑其实就是递归 AST 这颗树，然后将 静态节点 和 静态根节点 找到并打上标记

3. 代码生成器将ast转化正render函数字符串

<p title=”Berwin” @click=”c”>1

，生成后的代码字符串是：

with(this){return _c(‘p’,{attrs:{“title”:”Berwin”},on:{“click”:c}},[_v(“1”)])}格式化后：

with(this){
return _c(
‘p’,
{
attrs:{“title”:”Berwin”},
on:{“click”:c}
},
[_v(“1”)]
)
}

_c创建元素，_v创建一个文本节点，_s返回参数中的字符串
