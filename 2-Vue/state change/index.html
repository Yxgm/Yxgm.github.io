state change

一个button按下，页面上的state发生改变，vue在这里都干了什么

  1. 模版语法转化ast再转化render函数字符串，执行获得vdom，缓存hander；最终调用原生的addEventListener，绑定dom，对于v-for的遍历有一定的优化，即绑定的lister都是一个引用。（事件绑定原理）

  2. 初始化vue，进行数据劫持，即数据响应式，维护currentWatch唯一，收集依赖副作用函数（renderWatcher，或者其他watch），vue2递归，vue3惰性；（三种watcher）

  3. 赋值的时候，触发数据的setting，即发布出去订阅的副作用函数，但是并不知直接调用，而是走vue的异步更新策略，即调用dep.notify，通知收集的watcher去执行update函数，优雅降级，扔到浏览器micro tasks队列中，目的是为了批量更新数据，微任务插队机制，减少重排重绘。（nextTick）

  4. 数据变化之后，生成新旧vdom树，开始diff找出最小差异，遵循vue的diff策略（其中涉及到key的应用），然后patch将虚拟dom的差异应用到真实dom上，改变数据改变，但还没有渲染。

  5. 等待Event Loop自动执行更新函数，一轮tick结束，浏览再次渲染，新的一帧打在页面上。

  6. 同时在内部埋下的生命周期回调钩子，会在他该执行的时候执行。
