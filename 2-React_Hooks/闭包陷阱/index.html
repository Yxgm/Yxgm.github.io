闭包陷阱

useCallback

​ 我们一切照常的使用memo和useCallback给子组件传递函数，并且给useCallback设置上应该的依赖项。可是失去了useCallback函数的意义，每次都会刷新重置缓存函数。上技师！！

​ 虽然我们给useCallback的依赖函数加上count依赖项，但是重新刷新函数，这失去了useCallback缓存函数的意义。useCallback(fn , deps)相当于 useMemo(() => fn, deps)。

useRef

​ useRef可以解决这个闭包陷阱的问题，拿到的是导致闭包陷阱的第一次state，那么我们就修改第一次state即可。也就是每次返回相同引用的state。使用useRef不会出发page的rerender，有点骚操作。

​ 返回相同的state，react是不会刷新组件的，也就是利用了这个特性。是一个道理的

useEvent

​ react新推出的hook，为了解决这个问题。React团队认为这个Hook的主要应用场景是：「封装事件处理函数」。

​ useEvent，将传入的回调函数，用useRef保存，引用永远不变，同时将回调函数推迟到微任务（useLayoutEffect）中执行。也就意味着函数引用不变（ref保存），同时可以拿到最新的数据（微任务）

​ useEvent 总结一下

  1. 保证函数引用不变，同时还可以拿到最新的state，解决闭包陷阱，
  2. 实现，使用 ref 保证引用确定， 使用 useLayoutEffect 保证稳定赋值稳定（不会造成在 concurrent 重复执行）

函数组件的props跟新不及时问题：延迟输出props，如果在延迟期间props改变，函数组件无法正常或得到最新props。是由于函数组件方法内一直捕捉到的事闭包变量，并不是一个引用

如何解决：

  1. 子组件创造ref，手动管理ref，（改变父组件状态的同时，也给ref赋值）
  2. 类组件并没有这中现象，一直都是拿到的最新的porps。
