闭包和作用域

作用域->作用域链->闭包->内存监控

  1. javascript是一种编译型的语言。运行一段 javascript 代码，需要JS引擎和编译器。我们写的js代码经过词法/语法分析，（token流）代码生成机器码
  2. 引擎拿到了机器指令，接下俩LHS (Left-hand Side)（写值）和 RHS (Right-hand Side)（读值），二者区别就是对变量的查询目的是 变量赋值 还是 查询 。引擎想对变量进行操作就离不开他俩，LHS和 RHS获取变量的位置就是作用域。
  3. 作用域指程序中定义变量的区域，它决定了当前执行代码对变量的访问权限,同时作用域可以嵌套，即形成了作用域链（形成了变量的写/读的寻找路线）。
  4. 词法作用域是跟着他俩后面的概念，也成静态作用域，即捕捉变量的时候在定义函数的时候就已经确定，和哪里执行没有关系。
  5. 接下来块级作用域，简单来说，花括号内 {...} 的区域就是块级作用域区域。javascript 中大部分情况下，只有两种作用域类型：全局作用域 和 函数作用域，javascript 不是原生支持块级作用域的，起码创作者创造这门语言的时候压根就没把块级作用域的事情考虑进去。
  6. 但是ES6标准提出了使用let和const代替var关键字，来“创建块级作用域”。见作用域.md

闭包

一个大佬说的：闭包是“函数是第一公民，可作为参数和返回值传递”以及“函数可以定义在另一个函数里”这些特性导致的自然结果而已。

js中函数是一等公民

  1. 而闭包是支持头等函数编程语言中，实现词法绑定的一种技术。
  2. 在实现上，是一种结构体，它储存了一个函数和一个关联的环境变量。
  3. 闭包和函数最大的一个区别在于，在捕捉闭包的时候，他的自由变量会在补充时被确定，
  4. 即使是脱离了上下文，也可以正常运行。

闭包是什么

​ 一个函数和对其周围状态（lexical environment，词法环境）的引用捆绑在一起（或者说函数被引用包围），这样的组合就是闭包（closure也就是说，闭包可以在一个内层函数中访问到其外层函数的作用域，同时在外层环境能够访问其他函数内部变量的函数； 在 JavaScript 中，每当创建一个函数，闭包就会在函数创建的同时被创建出来；广义的来讲：javascript中的函数都是闭包，狭义的角度：javascript中的一个函数，访问了外层作用域的变量，那么它就是一个闭包。

闭包的应用

​ vue静态节点标记，react hook 闭包陷阱， 函数柯西化，发布订阅回调队列，redux中间件（闭包参数传递），单例，模拟私有变量，箭头函数的this绑定。只要我们想持久化一个变量，并且不被访问（开闭原则）就都可以使用闭包。

闭包的内存泄漏

​ 但当函数内再返回一个函数时，由于返回的函数持有外部函数的词法环境，而返回的函数又被其他生命周期东西所持有，导致外部函数虽然执行完了，但内存却无法被回收所以，返回的函数，它的生命周期应尽量不宜过长，方便该闭包能够及时被回收

​ 正常来说，闭包并不是内存泄漏，因为这种持有外部函数词法环境本就是闭包的特性，就是为了让这块内存不被回收，因为可能在未来还需要用到，但这无疑会造成内存的消耗，所以，不宜烂用就是了。

总结

  1. javascript 语言层面只原生支持两种作用域类型：全局作用域 和 函数作用域 。全局作用域程序运行就有，函数作用域只有定义函数的时候才有，它们之间是包含的关系。
  2. 作用域之间是可以嵌套的，我们把这种嵌套关系称为 作用域链。
  3. 可执行代码在作用域中查询变量时，只能查询 本地作用域 及 上层作用域，不能查找内部的函数作用域。JS 引擎搜索变量时，会先询问本地作用域，找到即返回，找不到再去询问上层作用域…层层往上，直到全局作用域。
  4. javascript 中使用的是 “词法作用域”，因此函数作用域的范围在函数定义时就已经被确定，和函数在哪执行没有关系。
  5. 有权访问另一个函数内部变量的函数，我们称为 闭包。闭包的本质是利用了作用域的机制，来达到外部作用域访问内部作用域的目的。
  6. 闭包的使用场景非常广泛，然而过度使用会导致闭包内的变量所占用的内存空间无法释放，带来 内存泄露 的问题。
  7. 我们可以借助于 chrome 开发者工具查找代码中导致了内存泄露的代码。Performance+Memory
  8. 避免内存泄露的几种方法：避免使用全局变量、谨慎地为DOM 绑定事件、避免过度使用闭包。最重要的，还是代码规范。 😃

js的内存泄漏

字面上的意思，申请的内存没有及时回收掉，被泄漏了

虽然前端有垃圾回收机制，但当某块无用的内存，却无法被垃圾回收机制认为是垃圾时，也就发生内存泄漏了

从根对象开始寻找，只要是根没有指向一个对象，垃圾回收机制就应该将其回收

上面是发生内存泄漏的根本原因，直接原因则是，当不同生命周期的两个东西相互通信时，一方生命到期该回收了，却被另一方还持有时，也就发生内存泄漏了
