堆栈、栈帧、string

string虽然是基本类型但是是放在堆里的

​ 我们知道v8引擎需要用执行栈来维护程序执行期间的上下文状态，而要想通过栈这种数据结构达到最高效率的切换上下文，执行栈的每一项都可以用相同长度的位来存储，只有这样JavaScript引擎才可以通过执行栈的索引来快速切换上下文，以提高程序的效率。

​ 所以v8引擎中的栈存储的都是相同位数的值，JavaScript引擎在遇到声明语句时，会在栈中申请一个空间，当给这个变量赋值时，会有两种情况：

  1. 如果值的类型是Boolean、Null、Undefined、Number，那么直接把这个值保存在栈中
  2. 如果是其他类型的值，JavaScript引擎会在堆中申请一个内存空间，然后把这个值保存在堆中，再将这个值在堆中的引用赋值给这个值在栈中的内存空间

​ 栈的结构和非常适合函数调用过程。因为函数可以被其他函数调用，子函数在被父函数调用的时候，父函数必定在子函数调用结束之后才有可能结束，因此在函数的声明周期上符合后进先出；另外由于函数有作用域，因此在资源分配的角度也符合后进先出

​ 在栈上分配资源和销毁资源的速度非常快，这主要归结于栈空间是连续的，分配空间和销毁空间只需要移动下指针就可以了，系统会自动释放。而堆是无序的，不会自动释放,栈中存储的每一项都是相同长度的引用，字符串可以无限长度，当然不是存在栈中

由于效率的原因，我们希望JS只复制对字符串的引用，而不是字符串的内容。但是另一方面，字符串在许多方面都和基本类型的表现相似，而字符串是不可变的这一事实（即没法改变一个字符串值的内容），因此可以将字符串看成行为与基本类型相似的不可变引用类型。

​ 对于string比较，比较的是堆中值的asico转化后的结果

​ 对于字符串，要是引用了s的属性，js就会价格字符串通过new String的方法转化为对象，包装成对象，当属性引用结束，这个新对象就会销毁这个对象。

​ 如果是new String（）的话就不会这样，new 出来的对象会持续保存，这也是原因所在。

栈和堆

栈：基本类型 （Undefined、Null、Boolean、Number）——原始值（原始值代表了原始的数据类型） 基本类型在内存中占据空间小、大小固定，他们的值保存在栈(stack)空间，上下文切换速度快，栈stack为自动分配的内存空间，它由系统自动释放；

内存中栈区的数据，在函数调用结束后，就会自动的出栈，不需要程序进行操作，操作系统会自动执行，换句话说：栈中的变量在函数调用结束后，就会消失。

因此栈的特点：轻量，不需要手动管理，函数调时创建，调用结束则消失。

堆：引用类型 （Object，Function，Array等）——引用值,引用类型占据空间大、大小不固定, 栈内存中存放地址指向堆(heap)内存中的对象。是按引用访问的,堆heap是动态分配的内存，大小不定也不会自动释放。

堆内存与栈内存是有区别的：栈内存运行效率比堆内存高，空间相对推内存来说较小，反之则是堆内存的特点。

1. 对js中闭包变量存储位置的思考

通过count调用之后，返回的函数中持有nums变量，由外部的变量持有。如果说基本类型在栈中，那么在count函数执行完之后，num就该销毁了，但是我们依然可以正常的去访问输出num，所以闭包num并没有销毁，他不在栈中，而是在堆中。

对于函数中的闭包变量，（return出去，被外部持有了生命周期），这些变闭包量会存在函数的scope属性上，是函数的一个属性[[Scopes]],因为这个闭包本身就是函数所产生的。

2. 变量类型

局部变量（函数中声明且在函数结束不会被其他作用域所使用）、被捕捉变量（被别的作用域变量使用）、全局变量。

var变量相当于给global对象添加了一条属性，而let/const变量不会改变windows对象，而是将变量的声明放在了一个特殊的对象下（与scope类似）。全局上下文是每一个函数闭包

3. 得出以下结论⚠️⚠️

  1. 被捕捉变量(函数捕捉到的闭包变量、全局变量)：值都是在堆中，栈中存地址。

  2. 局部变量(函数中声明的变量)：基本类型在栈，存储的是数据本身，引用类型地址放在栈，值放在堆。

捕捉变量全都存在堆中！唯一存在栈中的数据是局部变量中的未捕获的基础类型变量

总结：

全局中的基本类型和引用类型都存在堆中，地址存在全局执行上下文栈

局部变量的基本类型存在栈中、引用类型的地址存在栈中，值存在堆中、闭包的值存在堆中。

4. js如何判断一个变量是局部变量呢

​ 判断是否被内部函数引用即可

​ 仅有在函数体内的未被捕获变量是存在栈中（包括基础类型和引用类型的地址值）还有，在前期，js 的所有变量都是存在堆中的，那为什么有一部分变量到后来存在栈中呢？是因为编译器的优化，因为栈中访问更快。

​ 但如何判断出什么是局部变量，是一个难题，所以在早起的js 引擎中，所有的变量都是存储在堆中，直到发展到后期，引擎能判断出什么是局部变量，这才将局部的基本数据类型放在栈中，因为栈更符合函数执行的逻辑和更高的性能，这算是一种性能上的优化！

5. 栈帧概念

​ 调用栈是解释器追踪函数执行流的一种机制，可以通过调用栈追踪到哪个函数正在执行，而栈帧是指一个函数调用单独分配的那部分栈空间，有当前帧，和调用帧两个概念，函数A调用了函数B，当前这个情况，B是当前帧指栈顶，也就是新函数，调用帧指A函数，也是旧函数。函数执行完，会清除当前帧出调用栈。栈满，会引发堆栈爆炸。

​ 尾调用：在一个函数最后调用别的函数，即为尾调用，但是调用栈会不断的增多，最后一并弹出栈，可尾调用优化，即在最后调用别的函数的时候，加上return，即可结束当前函数，弹出当前函数的栈帧，当前调用栈只有two函数的栈帧了，进入下一函数的调用，对于chorme浏览器禁止了尾调用优化，safari是好的。

​ 尾递归：使用尾递归优化时，展开看到的调用栈中只会有当前执行的函数，不会储存上层的数据，这样就会减少内存的使用。 尾递归的本质实际上就是将方法需要的上下文通过方法的参数传递进下一次调用之中，以达到去除上层依赖。栈的使用空间为O(n)。
