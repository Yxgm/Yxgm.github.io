AOP思想

AOP

​ 面向切面编程，是一种非侵入式的扩充对象，方法和函数性能结合的技术。主要功能把一些核心业务逻辑无关的功能抽离出来，给原函数加一层，而不管原函数内部的实现。

​ Koa洋葱模型本质是AOP的面相切面编程，切面由中间件机制实现，中间件的单一指责，遵循先进后出的切面执行顺序，类似入栈出栈的顺序。

​ HTTP 切面流程（请求=>中间件=>响应） 从HTTP请求拿到想要的数据，然后处理想要处理的事情，最后返回处理后的结果

思路是一样的，将外层的函数，通过数组一对一对传入，然后遍历数组中的initialize或者close方法

AOP思想处处可见，就是在函数原来已有的功能上在加上一些功能。

  1. Vue2 的数组重写的7种方法，在真正push之前，触发一次更新
  2. react的事务机制，相当于before，after。react的HOC。
  3. redux的中间件处理，compare函数
  4. axios拦截器
  5. koa的鉴权，日志记录等等

装饰器和AOP

只是说装饰器的写法更加简洁，主要还是利用函数柯西化

总结

可以实现aop的方法：1. perform将中间件数组传递 2. 函数柯西化 3. 高阶函数 4. promise链式调用
