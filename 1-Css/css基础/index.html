animation
  1. animation-name：动画函数
  2. animation-duration:持续时间
  3. animation-timing-function：动画时间函数
  4. animation-delay：动画延迟运行时间
  5. animation-iteration-count：循环次数（infinite无限循环）
  6. animation-direction：CSS 属性指示动画是否反向播放（reverse ）
  7. animation-fill-mode ：设置 CSS 动画在执行之前和之后如何将样式应用于其目标
  8. animation-play-state：当前动画播放状态（running/paused）
过度transition

transition属性是 transition-property（变化属性），transition-duration（时间）、transition-timing-function（时间函数 ease-in, ease-out ease-in-out ）、transition-delay（延迟时间，延迟XXms之后执行过渡）的简写形式

一般配合hover或者active，一些伪元素使用，实现一些状态的改变。 eg： transition = “all 1s ease”

5. flex布局

​ 子元素属性：主轴方向上的实际尺寸是根据元素自身尺寸、flex-basis、flex-grow、flex-shrink 等属性共同决定的

  1. flex-grow：默认情况下不会成长，默认值是 0（子元素的 flex-grow 总和的值（>=1）平均分割父元素的剩余空间，增长的宽度=子元素的自身 flex 值/总和 flex-grow * 剩余空间） 小于 0 的时候，数值*剩余空间的宽度=子元素成长的宽度。

  2. flex-shrink：默认情况下就会收缩，默认值是 1，大于 1 时，数值* (剩余空间宽度/shrink 的总和）= 子元素收缩的宽度，自身宽度-收缩宽度=最终宽度，小于 1 的时候，数值*父元素宽度=子元素收缩的宽度。

  3. flex-basis属性定义了在分配多余空间之前，项目占据的主轴空间（main size）。

    1. 10%、写成百分比的时候代表占比，也就是单个item在container的占比大小。
    2. auto、代表根据item自身的width/height值来控制大小，也就是完全根据子列表项自身尺寸渲染。
    3. 100px、写数值的情况下 ，就是和写width：100px; flex-basis: auto没有区别，也就是基础长度
    4. context、由内容决定宽度

  4. 对flex-basis的一些总结

  1. flex布局中，子元素显示设置宽度或者高度是没有作用的，除非flex-basis设置为auto；
  2. flex-basis的优先级高，是content size suggestion的影响；
  3. flex-basis默认作用在content box上，这个和width属性是一样的，IE11会忽略box-sizing;
  4. flex-basis使用得当可以实现类似min-width/max-width的效果，配合grow和shark使用
  5. flex-basis数值属性值和width数值属性值不要同时使用，就不会遇到浏览器发现差异的问题。说的更极端一点，在Flex布局中，除非万不得已，否则没有使用width属性的理由，请使用flex-basis代替

⚠️ flex-basic设置为0px 和0% 是不同的，如果一个元素的 flex-basis属性 的值为百分数，且它父级元素（flex容器）在主轴方向上的尺寸没有被显式设置，此时 flex-basis 的值会被默认解析为 content。此种情况下，0% 呈现的结果会与 0px 不同。

  1. 子元素order属性：在主轴上的排列顺序，默认是0(往靠近主轴开始的方向向前排)，

11. 选择器解析

简言之就是 浏览器css匹配核心算法的规则是以 right-to-left 方式匹配节点的。
这样做是为了使规则能够快、准、狠地与render树上的节点匹配，通俗地将就是 就近原则。

试想一下，如果采用 left-to-right 的方式读取css规则，那么大多数规则读到最后（最右）才会发现是不匹配的，这样会做费时耗能，最后有很多都是无用的；而如果采取 right-to-left 的方式，那么只要发现最右边选择器不匹配，就可以直接舍弃了，避免了许多无效匹配。

显而易见，right-to-left 比 left-to-right 的无效匹配次数更少，从而匹配快、性能更优，所以目前主流的浏览器基本采取right-to-left的方式读取css规则。

12. 鼠标坐标

一、clientX、clientY
点击位置距离当前body可视区域的x，y坐标。
可以理解为距离浏览器窗口的距离，但注意这里不包括浏览器的导航栏距离只是浏览器内容区域。

二、pageX、pageY
对于整个页面来说，包括了被卷去的body部分的长度
相对于文档边缘,包含滚动条距离

三、screenX、screenY
点击位置距离当前电脑屏幕的x，y坐标

四、offsetX、offsetY
相对于带有定位的父盒子的x，y坐标
