文档流和文本流

文档流 normal flow

​ 指语言文本从左到右，从上倒下的展示，传统的HTML文档布局，平时我们所的BFC和IFC都是“Normal flow”，然而浮动(float)、绝对定位(absolute)、固定定位(fixed)三种方式定位会脱离文档流，但是float浮动是没有脱离文本流的，文档流是相对于盒子模型来讲的，元素浮动后，会让它跳出文档流，也就是说后面还有元素的时候，其他元素会无视它所占据的区域，直接在它的身下布局，但是文字却认同浮动元素所占据的位置，围绕它布局，也就是没有脱出文本流。

文本流 text flow

​ 是相对于文字段落讲的，对于absulute和fixed的定位方式是既脱离了文档流又脱离文本流。但是使用绝对定位后，不仅元素盒子会脱离文档流，就连下方的文字也会脱离文本流，那么后面的元素的文本就不会在认同它的位置，后直接在它后面布局，不会环绕。当然可以使用图层操作z-index ，这就是图层的概念。设置z-index的时候，只有设置了position属性：absolute/fixed等等才会生效

对于脱离文档流的盒子，样式改变不会引起重排，重绘
